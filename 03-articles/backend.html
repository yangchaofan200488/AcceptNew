后端开发简介

后端 指的是运行在后台并且控制着前端的内容，它主要负责程序设计架构思想，管理数据库等。后端更多的是应用到数据库并且进行交互以处理相应的业务逻辑。它需要考虑的是如何实现功能、数据的存取、平台的稳定性与性能等方面。它涉及到的动态语言如PHP、ASP、JSP等。

后端开发 是软件开发中的一个重要领域，专注于构建和维护那些运行在服务器上的应用程序部分，这些部分处理数据逻辑、业务规则、数据库交互以及为前端提供数据和服务。

推荐资源

  • LeetCode

    一个题库，里面收集了最常见、最高频的编程题目，每道题都有详细的解答，有助于提升算法能力。

  • GeeksForGeeks

    一个专门研究计算机科学的印度百科全书，包含编程、算法、面试问题等内容，适合从基础到专家级的学习。

  • 《MySQL必知必会》

    MySQL入门经典书籍，内容简洁明了，没有防自学机制。

  • 《labuladong算法小抄》

    适合算法初学者，提供了大量的算法技巧和解题思路。

常用工具

  • Visual Studio Code

    由微软开发的轻量级但功能强大的代码编辑器，支持多种编程语言和丰富的插件扩展，如Git集成、智能代码补全、调试工具等。VS Code的跨平台特性和可扩展性使其成为后端开发者的首选之一。

  • IntelliJ IDEA

    专为Java设计，但同样支持其他语言（如Kotlin、Groovy等）的强大IDE。它提供了高级编码帮助、重构工具以及对Spring和Hibernate等流行框架的内置支持，非常适合Java后端开发者。

  • Git与Github

    git是分布式版本控制系统，用于跟踪和管理代码的变更。Git提供了灵活性、速度和可靠性，是后端开发中进行版本控制和团队协作的必备工具。Github是基于Web的Git存储库托管服务，提供了用户友好的界面、问题跟踪和拉取请求功能，方便开发人员在项目上无缝协作。

  • Swagger

    用于设计、构建和记录RESTful API的开源工具。Swagger提供了一组丰富的工具来创建交互式API文档、生成客户端SDK和测试API端点，简化了API开发流程并增强了开发人员和利益相关者之间的协作。

  • Postman

    与Swagger作用类似 版本较旧 看个人习惯使用
